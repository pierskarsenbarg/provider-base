## What This Is

A boilerplate/template for building Pulumi native providers in Go using `pulumi-go-provider` (the `infer` package). It ships with a single example `Account` resource and `GetAccount` function to illustrate the patterns.

## Commands

This project uses [Task](https://taskfile.dev) (`task`) as the build runner. Key tasks:

```sh
task build_provider   # build the provider binary to ./bin/
task get_schema       # build and extract schema.json
task build_sdks       # build all language SDKs (node, dotnet, go, python)
task ensure           # go mod tidy for provider and sdk
task clean            # remove ./bin/
task watch            # rebuild provider on Go file changes
```

Individual SDK tasks: `task nodejs_sdk`, `task dotnet_sdk`, `task go_sdk`, `task python_sdk`.

Build manually without Task:
```sh
go build -o bin/pulumi-resource-base .
```

Versioning uses `pulumictl` (`pulumictl get version`).

## Architecture

### Provider implementation (`pkg/` + `main.go`)

- `main.go` — wires up the provider using `infer.Provider`. Registers resources, functions, config, and SDK language metadata. The `ModuleMap` remaps the `pkg` Go package to the `index` Pulumi module.
- `pkg/config.go` — provider config struct (`Config`). Implements `infer.CustomConfigure` to set defaults. Fields tagged `provider:"secret"` are treated as secrets.
- `pkg/account.go` — example resource (`Account`) and invoke function (`GetAccount`). Each resource needs three types: the resource struct, `Args` (inputs), and `State` (outputs). Implement CRUD methods (`Create`, `Read`, `Update`, `Delete`, `Diff`) on the resource struct pointer.

### Generated SDK (`sdk/go/base/`)

Auto-generated by `pulumi package gen-sdk`. Do not edit by hand — regenerate with `task go_sdk` after changing the provider schema.

### Pattern for adding a new resource

1. Create a new file in `pkg/` with three types: `Foo`, `FooArgs`, `FooState`.
2. Implement at minimum `Create` on `*Foo`; add `Read`, `Update`, `Delete`, `Diff` as needed.
3. Use `infer.Annotated` / `Annotate` methods to add schema descriptions.
4. Register the resource in `main.go`: `infer.Resource[*pkg.Foo, pkg.FooArgs, pkg.FooState]()`.
5. Rebuild the provider and regenerate SDKs.

### Pattern for adding a new invoke function

1. Define `GetFoo`, `GetFooArgs`, `FooResult` in `pkg/`.
2. Implement `Call(ctx, args) (result, error)` on `GetFoo`.
3. Register in `main.go`: `infer.Function[*pkg.GetFoo, pkg.GetFooArgs, pkg.FooResult]()`.

## Key Dependencies

- `github.com/pulumi/pulumi-go-provider` — the `infer` package drives schema generation and dispatch from Go struct tags and method signatures.
- `github.com/pulumi/pulumi/sdk/v3` — core Pulumi SDK (used in generated SDK code and codegen).
